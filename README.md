## 🚀 **QUICK START**

### **📋 Pré-requisitos**

```bash
# Versões mínimas
Node.js >= 18.0.0
npm >= 9.0.0
Docker >= 24.0.0
Git >= 2.40.0
```

### **⚡ Instalação Rápida**

```bash
# 1. Clone o repositório
git clone https://github.com/kryonix/kryonix-platform.git
cd kryonix-platform

# 2. Instale dependências
npm install

# 3. Configure ambiente
cp .env.example .env
# Edite o .env com suas configurações

# 4. Inicie banco de dados
docker-compose up -d postgres redis

# 5. Execute migrações
npm run db:migrate

# 6. Inicie desenvolvimento
npm run dev
```

### **🤖 Setup Ollama (IAs)**

```bash
# Instalar modelos essenciais (recomendado)
node scripts/setup-ollama-models.js essential

# Ou instalar modelos específicos
node scripts/setup-ollama-models.js coding      # Para programação
node scripts/setup-ollama-models.js lightweight # Para produção
node scripts/setup-ollama-models.js all        # Todos (200GB+)
```

### **🐳 Deploy com Docker**

```bash
# Deploy completo (todas as 25 stacks)
docker-compose -f docker-compose.production.yml up -d

# Ou deploy mínimo
docker-compose up -d
```

## 🐛 **TROUBLESHOOTING**

### **Problemas Comuns**

#### **🔧 Erro: "Cannot connect to database"**

```bash
# Verificar se PostgreSQL está rodando
docker-compose ps postgres

# Recriar banco se necessário
docker-compose down
docker volume rm kryonix_postgres_data
docker-compose up -d postgres
npm run db:migrate
```

#### **🤖 Erro: "Ollama models not found"**

```bash
# Verificar status do Ollama
curl http://localhost:11434/api/tags

# Reinstalar modelos essenciais
node scripts/setup-ollama-models.js essential
```

#### **⚡ Performance lenta**

```bash
# Verificar recursos
docker stats

# Limpar cache Redis
docker-compose exec redis redis-cli FLUSHALL

# Otimizar banco
npm run db:optimize
```

#### **🔐 Erro de autenticação**

```bash
# Verificar variáveis de ambiente
grep JWT_SECRET .env

# Regenerar JWT secret
openssl rand -hex 32
```

### **Logs e Debug**

```bash
# Logs da aplicação
docker-compose logs -f app

# Logs de stack específica
docker-compose logs -f n8n
docker-compose logs -f ollama

# Debug mode
NODE_ENV=development DEBUG=* npm run dev
```
